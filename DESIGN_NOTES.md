# ExamwaliSite Redesign Notes

## Current Website Problems

- The current implementation is effectively the default Vite starter, so it does not communicate what Examwali is, who it is for, or why a student should trust it.
- There is no exam-specific information architecture. A board student, an entrance aspirant, and a government exam learner all need different messaging and different next steps.
- The user flow is empty. There is no onboarding path, no product explanation, no conversion logic, and no meaningful CTA hierarchy.
- The visual system feels generic and unfinished, which weakens credibility for an education brand that should feel structured and dependable.

## Redesign Direction

- I reframed the website as "Examwali Mission Control" instead of a generic landing page.
- The hero now explains the product clearly, introduces a strong point of view, and lets users switch between exam tracks.
- The prototype adapts its content based on the selected track so the experience feels relevant instead of broad and vague.
- The layout uses a warmer, more editorial visual direction with dashboard cards, motion, layered gradients, and bold typography to avoid standard edtech sameness.

## Why These Decisions Improve UX

- Students can identify their lane quickly and understand whether the platform is relevant to them within seconds.
- The site now shows a believable study rhythm instead of making abstract promises.
- Mock tests, revision, and doubt-solving are presented as one connected system, which better matches how real exam preparation works.
- The experience has clearer hierarchy and stronger CTA flow, so the website is more persuasive and easier to navigate.
